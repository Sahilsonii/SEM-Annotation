The Project Scope

This project applies deep-learning object detection (YOLO11m) to Scanning Electron Microscope (SEM) images of perovskite thin-film solar cells. By automatically localizing defects — PbI₂ excess crystals, pinholes, and mixed-phase irregularities — we enable faster quality control in research labs and pilot production lines.

Traditional manual SEM annotation is time-consuming and subjective. Our pipeline replaces it with sub-second automated inference on 224×224 px image patches, achieving 42.19% mAP50 across 5 defect/background classes.

⚡

What Are Perovskite Solar Cells?

Perovskite solar cells (PSCs) use a crystalline material with the ABX₃ perovskite structure — typically methylammonium lead iodide (CH₃NH₃PbI₃) — as the light-absorbing layer. Key advantages:

  • Record lab efficiencies exceeding 26% (single-junction)
  • Solution-processable at low temperatures (<150°C)
  • Tunable bandgap via halide composition (1.2–2.3 eV)
  • Thin-film flexibility for building-integrated PV (BIPV)
  • Dramatically lower material and energy costs vs. silicon
🏭

Production & Scaling

Unlike silicon cells requiring 1400°C wafer processing, perovskites are manufactured via scalable coating techniques:

  • Slot-die coating — continuous roll-to-roll deposition
  • Inkjet printing — pixelated patterning for custom modules
  • Spray coating — large-area rapid deposition
  • Vacuum co-evaporation — high uniformity for tandems

The layered architecture (substrate → ETL → perovskite → HTL → electrode) is assembled sequentially, with laser scribing for monolithic series interconnection. Final encapsulation hermetically seals against moisture.

🔎

Defects in Perovskite Films

SEM imaging reveals critical microstructural defects that degrade cell performance:

  • PbI₂ Excess — Bright crystallites from incomplete precursor conversion; cause recombination losses
  • Pinholes (3D) — Dark voids in pure 3D perovskite; create shunt paths, reducing Voc
  • Pinholes (3D-2D mixed) — Voids at the 3D/2D interface; compromise passivation

Detecting these defects early in the fabrication pipeline prevents costly downstream failures.

🧠 YOLO11m Architecture

The detection pipeline processes images through an optimized backbone (darknet/CSP), a PANet neck for multi-scale feature fusion, and a decoupled detection head providing scale-specific outputs (P3, P4, P5).

🏗 Architecture Deep-Dive: YOLO11m

YOLO11m (Ultralytics, 2024) is a medium-sized single-stage object detector optimized for real-time inference. It processes images in a single forward pass through three stages: Backbone → Neck → Head.

🧠 Backbone — Feature Extraction

▼

The backbone extracts hierarchical features from the input image using convolutional blocks. YOLO11m uses an enhanced CSP (Cross-Stage Partial) architecture with C2f modules:

  • Conv2d + BatchNorm + SiLU — Initial stem block, reduces resolution by 2×
  • C2f (Cross-Stage Partial v2) — Splits feature maps, processes via bottleneck blocks, then concatenates for richer gradient flow
  • SPPF (Spatial Pyramid Pooling - Fast) — Multi-scale pooling at the backbone tail, capturing context at 5×5, 9×9, and 13×13 receptive fields

Output: 3 feature maps at ×8, ×16, ×32 downsampling (P3, P4, P5), encoding fine details through coarse semantics.

🔗 Neck — Feature Fusion (PANet)

▼

The neck fuses the backbone's multi-scale feature maps using a Path Aggregation Network (PANet):

  • Upsample + Concat — Top-down path brings high-level semantics to fine-grained features
  • C2f Fusion Blocks — Bottom-up path re-aggregates localization details from low levels
  • 1×1 Convolutions — Channel reduction for computational efficiency

This bidirectional fusion is critical for detecting small defects (pinholes ~10 px) alongside large PbI₂ clusters simultaneously.

📍 Detection Head — Decoupled Head

▼

YOLO11m uses a decoupled anchor-free head with separate branches for classification and regression:

  • Classification Branch — Predicts class probabilities for each grid cell (5 classes: PbI₂, 3D pinholes, 3D-2D pinholes, 3D background, 3D-2D background)
  • Regression Branch — Predicts bounding box coordinates (x, y, w, h) using Distribution Focal Loss (DFL) for precise localization
  • Objectness Score — Confidence that a grid cell contains any object, filtered at inference via NMS

The decoupled design improves convergence by letting classification and regression optimize independently.

⚙️ Training Configuration

▼
  • Epochs: 50
  • Image Size: 224 × 224 px
  • Dataset: Balanced (augmented class 2: 14→80 images, backgrounds subsampled)
  • Augmentation: Mosaic, MixUp, HSV jitter, random flip
  • Optimizer: AdamW with cosine LR scheduler
  • Loss: CIoU (bbox) + BCE (class) + DFL (distribution)
